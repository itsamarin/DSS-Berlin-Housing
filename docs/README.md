## Creating New Documentation

When adding new documentation:
1. Use Markdown format (.md)
2. Include table of contents for long documents
3. Add screenshots/diagrams where helpful
4. Keep language clear and concise
5. Update this README with new entries

## Documentation Standards

- **Headings**: Use # for main title, ## for sections
- **Code**: Use code blocks with language specification
- **Links**: Use relative paths for internal links
- **Images**: Store in `/docs/images/` folder
